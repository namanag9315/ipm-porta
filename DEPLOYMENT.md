# First-Time Free Hosting Guide (Step by Step)

This guide is for complete beginners.
You will deploy:
- **Backend (Django API)** → Render (free)
- **Database (PostgreSQL)** → Neon (free)
- **Frontend (React/Vite)** → Vercel (free)

---

## 0) What you need before starting

Create these accounts first (free plans are enough):
1. GitHub: https://github.com
2. Neon: https://neon.tech
3. Render: https://render.com
4. Vercel: https://vercel.com

Also install on your laptop:
- Git: https://git-scm.com/downloads

---

## 1) Put this project on GitHub

If your project is already on GitHub, skip to step 2.

If not:
1. Open terminal in your project folder.
2. Run:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   ```
3. Create a new empty repository on GitHub.
4. Connect and push (replace URL with your repo URL):
   ```bash
   git branch -M main
   git remote add origin https://github.com/<your-username>/<your-repo>.git
   git push -u origin main
   ```

---

## 2) Create free database on Neon

1. Login to Neon.
2. Click **Create Project**.
3. Choose any project name and region.
4. After project is created, open **Dashboard**.
5. Find **Connection string** (looks like `postgres://...`).
6. Copy it and save it in a notepad file. You will paste it in Render.

Important:
- Neon URLs often include SSL settings. Keep them as provided.

---

## 3) Deploy Django backend on Render

### 3.1 Create service from your GitHub repo
1. Login to Render.
2. Click **New +** → **Blueprint**.
3. Select your GitHub repo.
4. Render reads `render.yaml` and prepares a service named `ipm-portal-api`.
5. Click **Apply** / **Create**.

### 3.2 Add environment variables in Render
After service is created, open it and go to **Environment**. Set these:

- `DATABASE_URL` = (paste Neon connection string)
- `DJANGO_ALLOWED_HOSTS` = `<your-service-name>.onrender.com`
- `DJANGO_CSRF_TRUSTED_ORIGINS` = `https://<your-service-name>.onrender.com`
- `CORS_ALLOW_ALL_ORIGINS` = `False`
- `DJANGO_DEBUG` = `False`
- `GAS_BRIDGE_BASE_URL` = your Google Apps Script web app URL (for sheet sync)

`DJANGO_SECRET_KEY` is auto-generated by `render.yaml`.

If you use one direct payload endpoint instead of per-sheet IDs, set:
- `GAS_BRIDGE_URL` = your direct API URL

### 3.3 Deploy and verify
1. Click **Manual Deploy** → **Deploy latest commit** (or wait for auto-deploy).
2. Wait until status is **Live**.
3. Open this URL in browser:
   - `https://<your-service-name>.onrender.com/admin/`
4. If page opens, backend deployment is working.

### 3.4 Create admin user (for Django admin login)
In Render service:
1. Open **Shell** tab.
2. Run:
   ```bash
   python manage.py createsuperuser
   ```
3. Enter username/email/password.

Now login at `/admin/` with this account.

---

## 4) Deploy frontend on Vercel

1. Login to Vercel.
2. Click **Add New** → **Project**.
3. Import the same GitHub repo.
4. In project settings set:
   - **Root Directory**: `frontend`
   - **Build Command**: `npm run build` (usually auto-detected)
   - **Output Directory**: `dist`
5. Add environment variable:
   - `VITE_API_BASE_URL` = `https://<your-service-name>.onrender.com`
6. Click **Deploy**.
7. Open your Vercel URL after deployment.

Because `frontend/vercel.json` is included, refreshing routes should still work.

---

## 5) Connect frontend and backend correctly

After both are live:
1. Open frontend URL.
2. Try login/API features.
3. If requests fail with CORS/403:
   - Confirm Render env values are exact (no spaces).
   - Confirm `VITE_API_BASE_URL` points to Render backend URL (no trailing slash required).
   - Redeploy frontend after changing env vars.

---

## 5.1 First sync setup (IMPORTANT)

If sync is failing for the first time, usually one of these is missing:

1. **Sheet URLs are not saved in Admin Settings**
   - Login as IPMO admin.
   - Open Admin Settings page.
   - Fill and save:
     - timetable sheet URL
     - attendance sheet URL
     - (optional) mess menu sheet URL
     - (optional) birthday sheet URL

2. **GAS bridge URL is missing in Render env**
   - Add `GAS_BRIDGE_BASE_URL` (or `GAS_BRIDGE_URL`) in Render environment.
   - Redeploy after saving env vars.

3. **Running sync in async mode without Celery worker**
   - For first deployment, use `mode: "sync"`.
   - Use `async` only when Redis + Celery worker are configured.

## 6) What to do when you update code

Every time you push to GitHub:
1. Render redeploys backend (if auto-deploy enabled).
2. Vercel redeploys frontend.

Typical update flow:
```bash
git add .
git commit -m "your change"
git push
```

Then wait for Render + Vercel builds to finish.

---

## 7) Common errors and easy fixes

### Error: `DisallowedHost`
Fix `DJANGO_ALLOWED_HOSTS` on Render:
- Use exact domain like `your-service.onrender.com`

### Error: CSRF failed
Fix `DJANGO_CSRF_TRUSTED_ORIGINS`:
- Must include `https://your-service.onrender.com`

### Error: frontend can’t call API
- Check `VITE_API_BASE_URL` in Vercel.
- Redeploy frontend after changing env var.

### Error: database connection failed
- Recopy Neon `DATABASE_URL` carefully.
- Ensure full URL was pasted.

### Slow first request
- Render free plan sleeps when idle. First request may take time.

## 8) Optional local run (only if you want to test on laptop)

Backend:
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
```

Frontend (new terminal):
```bash
cd frontend
npm install
cp .env.example .env
# edit .env and set: VITE_API_BASE_URL=http://127.0.0.1:8000
npm run dev
```
