# Free Hosting Guide (Django API + React Frontend)

This project is split into:
- **Backend**: Django (`/` root)
- **Frontend**: Vite + React (`/frontend`)

Recommended free setup:
- **Backend** on Render (free web service)
- **Database** on Neon (free Postgres)
- **Frontend** on Vercel (free static hosting)

---

## 1) Deploy PostgreSQL (Neon free)

1. Create a Neon account: https://neon.tech
2. Create a new project/database.
3. Copy the connection string (`postgres://...`).
4. Keep it ready for Render as `DATABASE_URL`.

> Set `sslmode=require` in the URL if Neon shows it in the connection string.

---

## 2) Deploy Backend to Render

1. Push this repo to GitHub.
2. Login to Render: https://render.com
3. Click **New +** → **Blueprint** and select your repository.
4. Render will detect `render.yaml` and create service **ipm-portal-api**.
5. In Render service environment variables, set:
   - `DATABASE_URL` = your Neon Postgres URL
   - `DJANGO_ALLOWED_HOSTS` = `<your-render-service>.onrender.com`
   - `DJANGO_CSRF_TRUSTED_ORIGINS` = `https://<your-render-service>.onrender.com`
   - `CORS_ALLOW_ALL_ORIGINS` = `False`
   - `CORS_ALLOWED_ORIGINS` = `https://<your-vercel-app>.vercel.app`
   - `DJANGO_SECRET_KEY` (auto-generated by blueprint; you can keep it)
   - `GAS_BRIDGE_URL` = your Google Apps Script bridge URL (or use `GAS_BRIDGE_BASE_URL`)
   - Optional sheet fallbacks (IDs or full URLs):
     `TIMETABLE_SHEET_ID`, `ATTENDANCE_SHEET_ID`, `MESS_MENU_SHEET_ID`, `BIRTHDAY_SHEET_ID`
6. Trigger deploy.
7. After deploy, open:
   - `https://<your-render-service>.onrender.com/admin/`

### Create admin user
Use Render Shell:
```bash
python manage.py createsuperuser
```

---

## 3) Deploy Frontend to Vercel

1. Login to Vercel: https://vercel.com
2. Import the same GitHub repository.
3. Set **Root Directory** to `frontend`.
4. Build settings (auto-detected usually):
   - Build command: `npm run build`
   - Output directory: `dist`
5. Add env var:
   - `VITE_API_BASE_URL` = `https://<your-render-service>.onrender.com`
6. Deploy.

Because `frontend/vercel.json` is included, SPA routes will work on refresh.

---

## 4) Final production checks

- Backend health:
```bash
curl https://<your-render-service>.onrender.com/api/
```
- Frontend calls backend successfully from browser devtools/network.
- Admin login works.
- In IPMO admin panel, open Settings, paste your sheet URLs, and click Run Sync.

Optional manual sync (Render shell):
```bash
python manage.py run_sync
```

---

## 5) Local test before deploy

Backend:
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
```

Frontend:
```bash
cd frontend
npm install
cp .env.example .env
# Set VITE_API_BASE_URL=http://127.0.0.1:8000
npm run dev
```

---

## Notes

- Render free instances sleep when idle.
- If you need background Celery workers in production, add separate worker/Redis services (not included in the free minimal deployment above).
- Auto-sync runs in the web process by default. Tune with:
  `AUTO_SYNC_FALLBACK_INTERVAL_SECONDS` and `AUTO_SYNC_FALLBACK_INITIAL_DELAY_SECONDS`.
- Keep `DJANGO_DEBUG=False` in production.
